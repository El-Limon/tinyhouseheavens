### How Schemas Work

All JSON-LD is generated by `src/components/StructuredData.astro`. It takes a `type` prop and renders the appropriate schema. Currently supports: `organization`, `product`, `breadcrumb`, `faqPage`, `localBusiness`, `webSite`, `service`.

Usage in pages:
```astro
<StructuredData type="organization" locale={locale} />
<StructuredData type="product" locale={locale} product={modelData} />
```

### How Meta Tags Work

All meta tags (title, description, OG, Twitter, canonical, hreflang) are set in `BaseLayout.astro` via props passed from each page. SEO strings come from translation JSON files (`en.json`, `nl.json`, `fr.json`).

### How Blog Posts Work

Blog posts are MDX files in `src/content/blog/` with frontmatter:
```yaml
title: "Post Title"
description: "Meta description"
pubDate: 2026-04-10
author: "Tiny House Heavens"
tags: ["tag1", "tag2"]
locale: "en"
faqs:
  - question: "Question?"
    answer: "Answer."
```

The `faqs` array in frontmatter is used to generate FAQPage schema via StructuredData component.
